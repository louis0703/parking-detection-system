# 智能停车位检测与车牌增强系统 — 课程报告

---

## 一、项目背景与任务说明

### 1.1 项目背景

随着城市机动车保有量持续增长，停车难问题日益突出。智能停车管理系统能够自动检测车位占用状态、识别车牌信息，对于提高停车场管理效率、减少人工巡检成本具有重要的实际意义。

本项目是数字图像处理课程的综合实践项目，旨在将课程中学习的图像预处理、边缘检测、图像分割、形态学处理、频域滤波等经典算法应用于真实的停车场场景，构建一个完整的智能停车位检测与车牌增强识别系统。

### 1.2 任务说明

本系统包含两个核心任务：

**任务一：停车位占用检测**
- 输入：停车场俯视/前视图片
- 输出：每个车位的占用状态（empty / full）
- 要求：支持室内、室外网格地面、夜间等多种场景

**任务二：车牌区域增强与字符识别**
- 输入：包含车辆的场景图片
- 输出：车牌定位、增强后的车牌图像、识别出的车牌号码
- 要求：完成车牌检测、图像增强、OCR字符识别全流程

## 三、系统总体方案及流程图

### 3.1 系统架构

系统由两个并行的处理管线组成：

```
                    ┌─────────────────────────────────┐
                    │         输入图片                  │
                    └──────────┬──────────────────────┘
                               │
              ┌────────────────┴────────────────┐
              ▼                                  ▼
    ┌─────────────────┐              ┌─────────────────┐
    │  停车场图片处理   │              │  车牌图片处理     │
    └────────┬────────┘              └────────┬────────┘
             │                                │
    ┌────────▼────────┐              ┌────────▼────────┐
    │ 1.图像预处理     │              │ 1.车牌定位       │
    │ 2.边缘检测       │              │ 2.图像预处理     │
    │ 3.图像分割       │              │ 3.边缘检测       │
    │ 4.形态学处理     │              │ 4.车牌增强       │
    │ 5.频域处理       │              │ 5.字符识别(OCR)  │
    │ 6.车位状态判断   │              └────────┬────────┘
    └────────┬────────┘                       │
             │                        ┌───────▼───────┐
    ┌────────▼────────┐               │  识别结果输出   │
    │  empty / full   │               └───────────────┘
    └─────────────────┘
```

### 3.2 停车检测流程

**步骤1：图像预处理**
- 灰度化 → 直方图均衡化 → CLAHE自适应均衡 → 高斯去噪 → 中值去噪

**步骤2：边缘检测**
- Sobel / Prewitt / Roberts / Canny 四种方法对比

**步骤3：图像分割**
- 固定阈值分割 / Otsu自动阈值 / 连通域分析

**步骤4：形态学处理**
- 腐蚀 / 膨胀 / 开运算 / 闭运算

**步骤5：频域处理**
- DFT变换 → 理想/巴特沃斯/高斯低通滤波 → 高频增强滤波

**步骤6：车位状态判断**（双重检测机制）
- 检测方法A：棕色物体检测（HSV颜色空间 + 连通域分析）
- 检测方法B：纹理遮挡检测（块对比度分析 + 连通域分析）
- 综合判断：A 或 B 任一命中 → full，否则 → empty

### 3.3 车牌处理流程

**步骤1：车牌定位**
- 颜色定位（HSV蓝色/黄色掩膜 + 形态学 + 轮廓评分）
- 边缘定位（Canny + 形态学闭运算，备选方案）

**步骤2：图像预处理** — 同停车检测流程

**步骤3：边缘检测** — 同停车检测流程

**步骤4：车牌增强**
- 中值去噪 → CLAHE对比度增强 → 高频增强滤波（频域） → Unsharp Masking（空域锐化）

**步骤5：字符识别**
- RapidOCR 全图识别 + 区域匹配
- 文本后处理（省份纠错 + 大小写统一 + 字符纠错）

## 四、关键算法与方法说明

### 4.1 图像预处理

#### 4.1.1 灰度化
将彩色图像转换为单通道灰度图，采用加权平均法：`Gray = 0.299R + 0.587G + 0.114B`

#### 4.1.2 直方图均衡化
通过累积分布函数将灰度值重新映射，使直方图近似均匀分布，增强整体对比度。

#### 4.1.3 CLAHE（自适应直方图均衡化）
将图像分块（8×8），对每块独立做直方图均衡化，并限制对比度放大倍数（clipLimit=2.0~3.0），避免噪声过度放大。

#### 4.1.4 高斯去噪与中值去噪
- 高斯滤波：5×5高斯核平滑，去除高斯噪声
- 中值滤波：取邻域中值，有效去除椒盐噪声

### 4.2 边缘检测

| 算子 | 核大小 | 特点 |
|------|:------:|------|
| Sobel | 3×3 | 计算X/Y方向梯度，对噪声有一定抑制 |
| Prewitt | 3×3 | 均匀权重，简单快速 |
| Roberts | 2×2 | 最小核，对噪声敏感，边缘精细 |
| Canny | — | 多阶段：高斯平滑→梯度计算→非极大值抑制→双阈值连接 |

### 4.3 图像分割

- **固定阈值**：T=127，简单但不适应不同光照
- **Otsu自动阈值**：基于类间方差最大化自动选取最佳阈值
- **连通域分析**：对标记图提取连通区域，按面积过滤（>100px）

### 4.4 形态学处理

| 操作 | 核大小 | 作用 |
|------|-------|------|
| 腐蚀 | 5×5, 1次 | 消除小噪声点 |
| 膨胀 | 5×5, 2次 | 连接断裂区域 |
| 开运算 | 5×5 | 先腐蚀后膨胀，去除小物体 |
| 闭运算 | 25×5 | 先膨胀后腐蚀，填充小孔洞 |

### 4.5 频域滤波

通过DFT将图像变换到频域，应用滤波器后逆变换回空域：

- **理想低通滤波器**：截止频率处硬截断，有振铃效应
- **巴特沃斯低通滤波器**：阶数n=2，平滑过渡
- **高斯低通滤波器**：最平滑，无振铃
- **高频增强滤波器**：`H = a + b × (1 - H_GaussianLP)`，保留低频同时增强高频（cutoff=30, a=0.5, b=1.5）

### 4.6 车牌定位

**颜色定位法**：
1. 转HSV颜色空间
2. 创建蓝色掩膜（H=100-125, S>80, V>80）和黄色掩膜（H=15-35, S>80, V>80）
3. 形态学膨胀（3×3, 2次）+ 闭运算（25×5）连接字符
4. 轮廓检测 + 多条件筛选：
   - 长宽比 2.5~4.5（标准车牌≈3.14）
   - 面积：max(2000, 0.1%图像面积) ~ 5%图像面积
   - 最小高度50px，最小宽度120px
5. 综合评分：`score = ratio_score × position_score × area_penalty × area`

### 4.7 车牌增强

1. **中值去噪**：kernel=3，去除椒盐噪声
2. **CLAHE对比度增强**：clipLimit=3.0, tileGridSize=8×8
3. **高频增强滤波**：频域锐化，截止频率30
4. **Unsharp Masking**：`output = 1.5×original - 0.5×GaussianBlur`

### 4.8 车牌字符识别（OCR）

**引擎选择**：RapidOCR（基于PaddleOCR模型的ONNX推理），替代ddddocr

**识别策略**：全图识别 + 区域匹配
1. 对完整原始图像运行RapidOCR
2. 根据车牌区域坐标过滤，只保留中心点在区域内的文本
3. 容差设为区域尺寸的20%

**文本后处理**：
- 省份字符纠错（如"就"→"京"）
- 分隔符去除（"·"）
- 字母大写统一
- 常见字符纠错（i→1, o→0）

### 4.9 停车位占用检测（双重检测机制）

**方法A：棕色物体检测**
- HSV空间提取棕色像素（H=10-25, S>50, V>50）
- 形态学处理连接邻近区域
- 中心区域连通域分析，按面积和长宽比筛选紧凑物体
- 判据：最大紧凑物体面积 > 中心区域8% 且 长宽比 < 2.5 → 占用

**方法B：纹理遮挡检测**（针对车辆等非棕色物体）
- 将中心区域分成32×32像素块
- 计算每个块的对比度（max-min）
- 对比度 < 40 的块标记为"低对比度块"
- 闭运算连接相邻块 → 连通域分析
- 判据：最大连通域占中心区域5%~40% 且 整体亮度 > 80 → 占用

**综合判断**：方法A 或 方法B 任一命中 → full

## 六、参数调试与方法比较

### 6.1 CLAHE clipLimit 参数对比

| clipLimit | 效果 | 适用场景 |
|:---------:|------|---------|
| 2.0 | 对比度增强温和，保留细节 | 一般预处理 |
| 3.0 | 对比度增强较强，文字更清晰 | 车牌增强 |

### 6.2 边缘检测方法比较

| 方法 | 优点 | 缺点 | 本项目用途 |
|------|------|------|-----------|
| Sobel | 计算快，抗噪性好 | 边缘较粗 | 车位线条检测 |
| Prewitt | 实现简单 | 边缘定位不够精确 | 教学对比 |
| Roberts | 边缘精细 | 对噪声极敏感 | 教学对比 |
| Canny | 多阶段优化，边缘精细 | 计算量较大 | **主要使用** |

### 6.3 频域滤波器比较

| 滤波器 | 截止频率 | 特点 | 效果 |
|--------|:-------:|------|------|
| 理想低通 | 30 | 硬截断 | 有明显振铃效应 |
| 巴特沃斯LPF | 30, n=2 | 平滑过渡 | 较好，轻微振铃 |
| 高斯LPF | 30 | 最平滑 | 无振铃，但模糊 |
| 高频增强 | 30, a=0.5, b=1.5 | 增强高频 | **锐化效果最好** |

### 6.4 OCR引擎对比

| 引擎 | 1.jpg结果 | 2.jpg结果 | 10.jpg结果 | 准确率 |
|------|----------|----------|-----------|:------:|
| ddddocr | 就KBt355 ❌ | 5B929i2 ❌ | 粤a0hA88 ❌ | 0/3 |
| RapidOCR | 京KBT355 ✅ | 苏B92912 ✅ | 粤A0HA88 ✅ | 3/3 |

**分析**：ddddocr作为轻量级中文OCR模型，在车牌场景下存在省份字符误识别、大小写混淆、小尺寸车牌漏识别等问题。RapidOCR基于PaddleOCR的预训练模型，检测+识别一体化，在全图识别+区域匹配策略下表现优异。

### 6.5 停车检测阈值调试

| 参数 | 值 | 调试过程 |
|------|---|---------|
| 棕色HSV范围 | H=10-25, S>50, V>50 | 从H=10-35收紧到10-25 |
| 紧凑物体面积阈值 | 中心区域8% | 从5%提高到8%减少误检 |

## 七、失败样例与问题分析

### 失败样例1：ddddocr省份字符误识别

**现象**：1.jpg（京K·BT355）被ddddocr识别为"就KBt355"

**原因分析**：
- ddddocr模型将"京"误识别为"就"（字形相似）
- 小写"t"未转为大写"T"
- 模型对中文省份字符的识别不稳定

**解决方案**：
1. 换用RapidOCR引擎（基于PaddleOCR模型）
2. 添加省份字符纠错映射表
3. 字母自动大写化

**修复效果**：京KBT355 ✅

---

### 失败样例2：白色车辆未检测到

**现象**：10.jpg中白色五菱宏光停在车位上，但系统判断为"空闲"

**原因分析**：
- 原算法仅通过HSV颜色检测"棕色物体"（木板、施工材料）
- 白色车辆的H值不在棕色范围（H=10-25）内
- 白色车漆被完全忽略

**解决方案**：
1. 新增纹理遮挡检测方法
2. 将中心区域分成32×32块，计算局部对比度
3. 车漆面的块对比度低（均匀），空网格的块对比度高（亮砖+暗洞）
4. 低对比度块形成的大连通域 → 判为有车
